# Neotoma Developer Release Outreach Templates

Use these templates after survey segmentation.

Audience priority order:

1. Professional acquaintances with strong ICP fit
2. Former collaborators with strong ICP fit
3. Weak ties with likely fit
4. Low-fit contacts (passive updates only)

## Template A: Strong ICP (Direct Invite)

Subject: Neotoma developer release fit

Hey {name},

Your workflow profile looks like a strong fit for what we are testing in the Neotoma developer release.

Neotoma is focused on deterministic state for long-running agents: versioned transitions, replayability, and auditability. The goal of this release is to validate the memory invariant with real production-like workflows.

If useful, I can give you access and we can do:

- one real workflow test in your environment
- async feedback on where state integrity breaks down
- a short technical debrief

If you are in, I will send setup and a suggested first test scenario.

## Short DM Version (Strong ICP)

## Follow-Up Prompts for Accepted Testers

Use these after they agree:

1. What is the first real workflow you want to run?
2. Where does state currently drift or become unreproducible?
3. What would count as a successful first test for you?
4. Can we schedule a 20-30 minute debrief after your first run?
