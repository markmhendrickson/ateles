# Agent Decision-Making Framework

**Status:** Active  
**Last Updated:** 2025-01-15  
**Related:** `/shared/docs/agent-context.md`, `/strategy/operations/operating-manual.md`

---

## Purpose

This document provides a framework for agents to make decisions, create/update documents, and execute processes while maintaining consistency with repository principles and behavioral mandates.

---

## When Making Recommendations

1. **Check strategy first** - Ensure alignment with `/strategy/strategy/` documents for the relevant domain
2. **Review tactics** - Use established tactical approaches from `/strategy/tactics/` for the relevant domain
3. **Follow operations** - Execute via procedures in `/strategy/operations/[domain]/`
4. **Respect behavioral mandates** - Reference `/strategy/operations/operating-manual.md` for domain-specific rules
5. **Use established signals** - Prefer established frameworks over ad-hoc decisions

---

## When Creating/Updating Documents

1. **Maintain hierarchy** - Strategy > Tactics > Operations
2. **Preserve consistency** - Align with existing canonical documents
3. **Document decisions** - Update relevant strategy/tactics documents
4. **Save all outputs** - Scorecard results, quarterly reports, etc. (see `/shared/docs/agent-workflow-requirements.md`)
5. **Follow naming conventions** - Use YYYY-MM-DD format for dates (see `/shared/docs/agent-workflow-requirements.md`)

---

## When Executing Processes

1. **Follow established workflows** - Reference `/strategy/operations/[domain]/` for checklists and templates
2. **Save all results** - Scorecards, reports, analyses (mandatory for finance domain)
3. **Verify compliance** - Check behavioral rules adherence (reference `/strategy/operations/operating-manual.md`)
4. **Update documents** - Reflect changes in canonical documents

---

## Behavioral Compliance

**Reference:** `/strategy/operations/operating-manual.md` for complete behavioral rules (Daily, Weekly, Quarterly, Annual).

### Agent Requirements

- Verify compliance with behavioral mandates when executing processes
- Quarterly reviews verify adherence to all rules
- Respect domain-specific constraints (e.g., finance: no discretionary trading, quarterly-only cadence)

### Important Constraints

**Reference:** `/strategy/operations/operating-manual.md` for complete constraint documentation (Trading, Monitoring, Concentration, etc.).

**Agent Requirements:**
- Verify no violations of forbidden actions when executing processes
- Respect trading constraints (quarterly-only cadence)
- Respect monitoring constraints (no daily portfolio monitoring)
- Respect concentration constraints (reference `/strategy/tactics/stx-playbook.md` for STX limits)
